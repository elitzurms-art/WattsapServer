# תכנון: שאילה של פריטים משוריינים

## מטרה
לאפשר שאילה של פריט משוריין ע"י אדם אחר, תוך שמירת המידע על השריון.

## סטטוסים חדשים
- `מושאל` - פריט מושאל רגיל
- `משוריין` - פריט משוריין
- `מושאל+משוריין` - **חדש!** פריט מושאל כרגע + משוריין לעתיד

## מבנה נתונים

### פריט רגיל משוריין:
```
סטטוס: "משוריין"
שם: "דני"
טלפון: "972501111111"
תאריך תחילת שיריון: "15/02/2026"
תאריך סיום שיריון: "20/02/2026"
```

### פריט מושאל ומשוריין:
```
סטטוס: "מושאל+משוריין"
שם: "משה,דני"  (שואל ראשון, משריין שני)
טלפון: "972502222222,972501111111"
תאריך תחילת שיריון: "15/02/2026"  (של המשריין)
תאריך סיום שיריון: "20/02/2026"
```

## שינויים נדרשים בקוד

### 1. handlers/index.js - accept()
**שורות 703-739** - לוגיקת BORROW_CONFIRM

צריך לבדוק עבור כל פריט נשאל:
- אם הפריט משוריין על מישהו אחר (לא השואל)
- אם כן → לשנות סטטוס ל"מושאל+משוריין" ולהוסיף את השואל לתחילת הרשימה
- אם לא → להשאיר את הלוגיקה הקיימת

### 2. sheets/inventory.js - getAvailableItems()
**שורות 7-61**

צריך לזהות גם סטטוס "מושאל+משוריין" כזמין:
```javascript
if (status !== 'במלאי' && status !== 'משוריין' && status !== 'מושאל+משוריין') return;
```

אבל רק אם השואל הנוכחי הוא לא המשתמש.

### 3. sheets/inventory.js - getBorrowedItemsByPhone()
**שורות 67-131**

צריך לזהות גם "מושאל+משוריין":
```javascript
if (status !== 'מושאל' && status !== 'משוריין' && status !== 'מושאל+משוריין') return;
```

### 4. handlers/index.js - formatItem() בכל הפונקציות
צריך להציג נכון פריטים עם סטטוס משולב.

### 5. תגובות - הוספת שדה חדש?
לשקול אם להוסיף שדה "המשריינים שנותרו" כדי לעקוב אחרי מה קרה.

## זרימה מלאה

### שאילה:
1. משתמש בוחר פריט 305 (משוריין על דני)
2. בconfirmation רואה אזהרה: "פריט זה משוריין לדני 15/02-20/02, עליך להחזיר לפני כן"
3. מאשר
4. הקוד:
   - בודק שהפריט משוריין על מישהו אחר
   - מוסיף את השואל לתחילת הרשימה
   - משנה סטטוס ל"מושאל+משוריין"
   - שומר ב"תגובות"

### החזרה של השואל:
1. משה מחזיר את 305
2. הקוד:
   - מזהה שהסטטוס "מושאל+משוריין"
   - מוציא את משה מהרשימה
   - משנה סטטוס חזרה ל"משוריין"
   - דני נשאר עם השריון

### ביטול שריון:
1. דני מבטל את השריון של 305
2. הקוד:
   - מזהה שהסטטוס "מושאל+משוריין"
   - מוציא את דני מהרשימה
   - משנה סטטוס ל"מושאל"
   - משה נשאר כשואל
