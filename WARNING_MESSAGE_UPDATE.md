# עדכון: הודעות אזהרה לפריטים משוריינים

## 🎯 מה עשינו

הוספנו הודעות ברורות כשמשתמש **מחזיר** פריט שהוא גם **משוריין** על מישהו אחר.

---

## ✅ שינויים שבוצעו

### 1. inventory.js - חישוב תאריך החזרה חובה
**שורה 38 (ו-108)**

**לפני:**
```javascript
const reserveReturnBy = (status === 'משוריין' && firstReserveDate) ? getDayBefore(firstReserveDate) : null;
```

**אחרי:**
```javascript
const reserveReturnBy = ((status === 'משוריין' || status === 'מושאל+משוריין') && firstReserveDate) ? getDayBefore(firstReserveDate) : null;
```

**למה?** עכשיו גם פריטים עם סטטוס "מושאל+משוריין" מקבלים תאריך החזרה חובה.

---

### 2. handlers/index.js - handleReturnSelect
**שורה 273-275**

כשמשתמש רואה פריט שהוא שאל והוא גם משוריין למישהו אחר:

**אחרי:**
```javascript
return `• *${item.id}* - ${item.name} ⚠️ *הפריט משוריין ולכן חובה עליך להחזירו עד - ${item.reserveReturnBy}*`;
```

### 3. handlers/index.js - choice() (אישור סופי)
**שורה 513-523**

באישור הסופי, הוספנו בדיקה נפרדת לפריטים מושאל+משוריין:

```javascript
if (currentState === 'RETURN_SELECT') {
    regularItems.forEach(i => {
        // פריט משוריין על המשתמש
        if (i.myTo) {
            message += `\n⚠️ פריט ${i.id}: משוריין על שמך בין התאריכים ${i.myFrom}-${i.myTo}`;
        }
        // פריט מושאל+משוריין והמשתמש הוא השואל (בלי תאריכים)
        else if (i.status === 'מושאל+משוריין' && !i.myTo) {
            message += `\n⚠️ *פריט ${i.id}: הפריט משוריין ולכן חובה עליך להחזירו עד - ${i.reserveReturnBy}*`;
        }
    });
}
```

- רשימה: "דני, משה" (משריין קודם, שואל אחרון)

## ✨ יתרונות

1. **ברור ומודגש** - המשתמש רואה בבירור שיש חובה להחזיר עד תאריך מסוים
2. **פעמיים** - מופיע גם ברשימה וגם באישור, אי אפשר לפספס

## 📝 הערות טכניות

- התאריך המוצג הוא **יום לפני השריון** (אם השריון מ-10/03, אז החזרה עד 09/03)
- זה מחושב אוטומטית ע"י הפונקציה `getDayBefore()`
- הבדיקה היא: `i.status === 'מושאל+משוריין' && !i.myTo` (פריט מושאל+משוריין ואין תאריכים = המשתמש הוא השואל)
